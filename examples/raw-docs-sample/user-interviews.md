# 用户访谈记录

## 访谈对象：张三（独立开发者）

**日期：** 2026-02-15

**背景：** 正在开发一个 AI 写作助手

### 主要痛点

**Q: 你目前使用什么 AI API？**
A: 我用 OpenAI 的官方 API，但是太贵了。我的应用还在测试阶段，用户不多，但每个月 API 费用就要 $200+。

**Q: 你尝试过其他方案吗？**
A: 试过 OpenRouter，但发现价格比官方还贵。也看过一些国内的平台，但担心稳定性。

**Q: 如果有一个平台能解决这些问题，你最看重什么？**
A:
1. 价格要便宜，至少比官方低 30%
2. 稳定性要好，不能三天两头出问题
3. 迁移要简单，我不想重写代码

### 使用场景

- 每天调用 API 约 5000 次
- 主要使用 GPT-4 和 Claude
- 需要快速切换模型进行测试

### 预算

- 理想预算：$100/月
- 可接受上限：$150/月

## 访谈对象：李四（AI 创业团队 CTO）

**日期：** 2026-02-20

**背景：** 创业公司，团队 5 人，开发 AI 客服产品

### 主要痛点

**Q: 你们团队目前的 API 成本是多少？**
A: 每个月大概 $800-1000，占了我们运营成本的 40%。

**Q: 你们考虑过降低成本吗？**
A: 当然考虑过，但是：
1. 不敢用不知名的平台，怕影响服务稳定性
2. 迁移成本高，我们已经写了很多代码
3. 担心被单一供应商锁定，万一涨价怎么办

**Q: 如果有一个平台能解决这些问题，你会考虑吗？**
A: 会的，但我需要：
1. 看到真实的用户案例
2. 有详细的 SLA 保证
3. 支持多个模型，保持灵活性

### 决策因素

1. 价格（权重 40%）
2. 稳定性（权重 30%）
3. 迁移成本（权重 20%）
4. 技术支持（权重 10%）

---

## 关键洞察

### 用户真正关心的问题

1. **成本控制** - 不是"便宜"，而是"可预测的成本"
2. **迁移风险** - 不是"功能多"，而是"迁移简单"
3. **稳定性保证** - 不是"承诺"，而是"可验证的数据"

### 内容营销方向

1. 写一篇"从 OpenAI 迁移到 MyAPI 只需改一行代码"
2. 发布真实的成本对比数据
3. 展示稳定性监控数据（Uptime、响应时间）
4. 分享用户案例（降低了多少成本）
